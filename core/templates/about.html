About This Project: LMS Proyek

Project Overview

This project is a simple Learning Management System (LMS) built with Django. It allows teachers to create rooms, manage schedules, upload materials, and enroll students. Students can view their schedules and access learning materials.

Key Features

  • User Authentication: Secure login and registration for both teachers and students.
  • Role-Based Dashboards: Separate dashboards for teachers and students with relevant functionalities.
  • Room Management: Teachers can create and manage virtual classrooms.
  • Schedule Management: Teachers can create and view schedules for their rooms.
  • Material Upload: Teachers can upload various learning materials (e.g., PDFs) to their rooms.
  • Student Enrollment: Teachers can add students to their rooms.
  • Profile Management: Users can view their profile information.

Technical Details

The project is developed using the Django web framework, a high-level Python Web framework that encourages rapid development and clean, pragmatic design.

  • Backend: Python with Django.
  • Database: SQLite (default for development, easily swappable for PostgreSQL/MySQL in production).
  • Frontend: HTML templates rendered by Django, styled with Tailwind CSS (via CDN for simplicity).
  • User Management: Django's built-in authentication system, extended with a custom Profile model for role management (Guru/Siswa).
  • Forms: Django's powerful Form API for handling user input and model creation.
  • Static and Media Files: Django's staticfiles and media handling for CSS/JS and user-uploaded content.
  • API Endpoints: Simple JSON responses for dynamic content (e.g., jadwal_detail_api).

Code Structure Highlights

  • core/models.py: Defines the database schema (Profile, Ruangan, Materi, Jadwal).
  • core/views.py: Contains the logic for handling web requests and rendering templates. Refactored into role-specific dashboards (guru_dashboard, siswa_dashboard) for better organization.
  • core/forms.py: Custom forms for registration, room creation, schedule/material creation, and adding students.
  • core/urls.py: Defines URL routing for the application.
  • core/templates/: HTML templates for various pages, extending base.html for consistent layout.
